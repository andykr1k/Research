* Ideas for a new loss function:
** The Functional-Semantic Loss (Hybrid)
*** Tools have two layers: what they say they do (documentation) and what they actually do (input/output schema).
*** The Idea: Create a joint loss that aligns the semantic embedding (from a README) with the functional embedding (from the JSON schema).
*** For batch item i, golld tool y_i:
**** e_i = normalized query embedding
**** p_j = normalized prototype for tool j
**** s_ij = e_i · p_j # cosine similarity
**** z_ij = alpha * s_ij # alpha = exp(logit_scale), clamped
*** Formula Logic: L_total = L_semantic + lambda * L_functional
*** L_semantic = (1/B) * sum_i -log( exp(z_i,y_i) / sum_j exp(z_i,j) )
*** w_ij = (1 - C[y_i, j]) * 1[j != y_i]
*** d_ij = 1 - s_ij
*** v_ij = max(0, m - d_ij)
*** L_functional = mean_i ( sum_j w_ij * v_ij^2 / sum_j w_ij)
*** C[a,b] is the compatibility score between tool a and tool b
*** lambda is --compatibility-weight
*** m is --compatibility-margin
*** Objective: L_total = CrossEntropy(query -> correct tool)+ lambda * weighted_margin_penalty(on incompatible close negatives)
*** This prevents the agent from confusing two "Weather" tools if one requires a Lat/Long and the other requires a Zip Code.

** Hierarchical Margin Loss
*** Since my data is "very similar," a fixed margin $m$ is too blunt. If I have a hierarchy (e.g., Search -> Web Search -> Google Search), the loss should enforce smaller distances for sub-categories and larger ones for different categories.
*** The Idea: Make $m$ a function of the taxonomic distance between tools.
*** Effect: Tools in the same family stay in a "neighborhood," but the loss still forces a gap so the agent can distinguish between them.

** Capability-Action Loss (The "Trajectory" approach)
*** For an agent, a tool is a transition in state.
*** The Idea: Instead of embedding the tool as a point, embed it as a transformation vector.
*** Implementation: Use a loss that minimizes $\| (State_{pre} + Tool_{embedding}) - State_{post} \|$. This is similar to how Word2Vec handles "King - Man + Woman = Queen," but for tool executions.
